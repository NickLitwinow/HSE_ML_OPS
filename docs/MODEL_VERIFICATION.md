# Model Verification Checklist (UV)

Перед выпуском новой версии модели (Release Candidate) необходимо выполнить следующие проверки.

## 1. Метрики качества (Quality Metrics)
- [ ] **Accuracy** на отложенной тестовой выборке > 0.85.
- [ ] **ROC-AUC** > 0.90.
- [ ] **Precision/Recall**: Баланс метрик соответствует бизнес-требованиям (например, Recall > 0.8 для выявления дефолтов).
- [ ] Сравнение с предыдущей версией: Новая модель не должна быть хуже текущей production-версии более чем на 1%.

## 2. Производительность (Performance)
- [ ] **Latency (P95)**: Время предсказания на 1 объект < 50ms.
- [ ] **Batch Processing**: Время обработки пакета из 1000 объектов < 200ms.
- [ ] **Throughput**: Сервис выдерживает нагрузку > 100 RPS на одном инстансе.
- [ ] **Размер артефакта**: Размер файла модели (`.pkl`) < 500MB.

## 3. Проверка данных (Data Validation)
- [ ] **Data Drift**: Распределение ключевых признаков (Income, Credit Score) не отличается от обучающей выборки (PSI < 0.1).
- [ ] **Schema Check**: Все обязательные признаки присутствуют, типы данных соответствуют ожидаемым.
- [ ] **Outliers**: Отсутствуют аномальные значения (например, возраст < 0 или > 120).

## 4. Безопасность и Стабильность (Security & Stability)
- [ ] **Missing Values**: Модель корректно обрабатывает пропущенные значения (не падает с ошибкой).
- [ ] **Adversarial Attacks**: Модель устойчива к небольшим возмущениям входных данных.
- [ ] **Reproducibility**: Обучение модели воспроизводимо при фиксации `random_state`.
- [ ] **Dependency Scan**: В зависимостях (`requirements.txt`) нет критических уязвимостей (проверка `safety check`).
